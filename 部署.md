# 🚀 阿里云香港 ECS 双项目部署指南（1Panel · 图形化）

全程在 1Panel 面板中操作，无需 SSH 客户端。

---

## 1. 服务器选购

阿里云控制台 → **ECS** → **创建实例**：

| 配置项 | 建议值 |
|--------|--------|
| 地域 | **香港** |
| 实例规格 | 2 vCPU / 4 GiB |
| 系统盘 | 40 GiB ESSD |
| 镜像 | Ubuntu 22.04 |
| 公网 IP | ✅ 分配 |

### 安全组配置

**安全组** → **配置规则**，添加入方向规则：

| 端口 | 来源 | 说明 |
|------|------|------|
| 22 | 0.0.0.0/0 | SSH（安装 1Panel 用一次） |
| 80 | 0.0.0.0/0 | HTTP |
| 443 | 0.0.0.0/0 | HTTPS |
| 面板端口 | 0.0.0.0/0 | 1Panel 入口 |

> 1Panel 安装时会告诉你面板端口（如 8090），记下来加到安全组。

## 3. 安装 1Panel（唯一一次 SSH）

```bash
ssh root@47.239.205.150
curl -sSL https://resource.fit2cloud.com/1panel/package/quick_start.sh -o quick_start.sh && bash quick_start.sh
```

安装完后输出面板地址、用户名、密码，**记下来**。后续全部操作在面板里完成。

---

## 4. 登录 + 环境准备

### 4.1 登录

浏览器打开 `http://47.239.205.150:<面板端口>`，输入用户名密码。

### 4.2 安装 Node.js

左侧菜单 → **工具箱** → **运行环境** → 点击 **安装** → 选择 **Node.js 20.x** → 确认。

等待几分钟安装完成。

### 4.3 安装 Chromium + pnpm

左侧菜单 → **主机** → **终端**，在网页终端中执行：

```bash
# pnpm
npm install -g pnpm

# Chromium（PDF 生成）
apt install -y chromium-browser
```

## 5. 部署项目一

### 5.1 获取代码

首选 git clone，在 1Panel 网页终端：

```bash
mkdir -p /opt
cd /opt
git clone <项目一仓库地址> project-a
cd project-a
pnpm install
```

> 💡 如果 git clone 因为网络原因失败，备选方案：在本地把项目打包（去掉 node_modules 和 client/dist），通过 1Panel **文件管理** 上传到 `/opt`，再在网页终端解压：
>
> ```bash
> # 本地打包
> tar --exclude='node_modules' --exclude='client/dist' --exclude='.git' -czf project-a.tar.gz .
> ```
>
> 上传后在服务器解压：
>
> ```bash
> cd /opt && mkdir project-a && cd project-a
> tar -xzf ../project-a.tar.gz
> pnpm install
> ```

### 5.2 构建前端

仍在网页终端：

```bash
cd /opt/project-a/client
pnpm build
```

> 构建产物在 `client/dist` 目录。

### 5.3 创建上传目录

```bash
mkdir -p /opt/project-a/server/uploads
```

### 5.4 创建网站

左侧 → **网站** → **创建网站** → **静态网站**：

| 字段 | 值 |
|------|-----|
| 域名 | `mmnas.icu` |
| 网站目录 | `/opt/project-a/client/dist` |

点击 **创建**。

### 5.5 添加反向代理

在网站列表中，点击刚创建的网站，进入它的设置页。

**反向代理** 标签 → **添加**：

| 路径 | 目标 |
|------|------|
| `/api` | `http://127.0.0.1:3001` |
| `/uploads` | `http://127.0.0.1:3001` |

> 这样前端页面里的 `/api/xxx` 请求就会被转发到后端。

### 5.6 启动后端

左侧 → **工具箱** → **进程守护** → **创建**：

| 字段 | 值 |
|------|-----|
| 名称 | `project-a-server` |
| 启动命令 | `node index.js` |
| 工作目录 | `/opt/project-a/server` |
| 进程数 | 1 |

展开 **环境变量**，添加：

```
PORT=3001
PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true
PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser
```

点击 **创建** → **启动**。

### 5.7 SSL 证书

网站列表 → 点击 `mmnas.icu` → **SSL** → **申请证书** → 勾选 Let's Encrypt → 确定。

> 证书自动续期，无需维护。

### 5.8 验证

浏览器访问 `https://mmnas.icu`。

### 6.4 验证

## 7. 更新部署

### 项目一（有后端）

代码有更新时，打开 1Panel 网页终端：

```bash
cd /opt/project-a
git pull
pnpm install
cd client && pnpm build && cd ..
```

然后在 **进程守护** 中点击 `project-a-server` 旁边的 **重启** 按钮。

### 项目二（纯静态）

```bash
cd /opt/project-b
git pull
```

Nginx 会自动加载新文件，无需重启。

---

## 8. 操作总结

| 步骤 | 在哪操作 |
|------|----------|
| 克隆代码 | 1Panel 网页终端 |
| pnpm install / build | 1Panel 网页终端 |
| 创建网站 | 1Panel 面板 UI |
| 反向代理 | 1Panel 面板 UI |
| SSL 证书 | 1Panel 面板 UI（一键） |
| 启动/停止/重启后端 | 1Panel 面板 UI（进程守护） |
| 浏览文件 | 1Panel 面板 UI（文件管理） |

**全程只需在初次安装时 SSH 一次，之后所有操作都在浏览器里完成**

## 9. 架构

```
浏览器
  │
  ├── https://mmnas.icu ──→ 1Panel/Nginx :80/:443
  │                               │
  │               Host: mmnas.icu
  │               → /         → /opt/project-a/client/dist/
  │               → /api/*    → http://127.0.0.1:3001
  │
  └── https://mmfnnas.asia ──→ 1Panel/Nginx :80/:443
                                  │
                  Host: mmfnnas.asia
                  → /         → /opt/project-b/（纯静态HTML）
```
